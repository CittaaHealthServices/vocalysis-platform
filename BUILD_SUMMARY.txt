✓ Processors (8 files) - /src/processors/
  1. audioAnalysis.js (261 lines)
     - Audio extraction via VocaCore /extract
     - Risk analysis via VocaCore /analyze
     - Alert evaluation and creation
     - Employee wellness profile updates
     - Email & webhook queue trigger
     - 100% error handling with try/catch per step

  2. pdfGeneration.js (195 lines)
     - PDFKit-based report generation
     - Saves to Railway volume storage
     - Contextual recommendations based on risk level
     - Professional formatting with Cittaa branding

  3. emailNotifications.js (222 lines)
     - 7 email templates (assessment_invite, alert_notification, etc.)
     - Nodemailer SMTP integration
     - Email hashing for privacy logging
     - 3-attempt retry with 30s backoff
     - 100/min rate limiting

  4. webhookDelivery.js (159 lines)
     - HMAC-SHA256 signature generation
     - HTTP POST with 10s timeout
     - Exponential backoff: 1min → 5min → 30min → 2hr → 6hr
     - Permanent failure after 5 attempts
     - WebhookDeliveryLog audit trail

  5. bulkImport.js (267 lines)
     - CSV parsing via csv-parse
     - Email validation, duplicate detection
     - Batch insert (50 records per batch)
     - Redis + MongoDB persistence
     - Summary email to HR Admin
     - Error tracking with first 100 errors logged

  6. scheduledAssessments.js (66 lines)
     - Cron: Daily 8:00 AM IST
     - Finds employees with nextScheduledDate = today
     - Queues assessment_invite emails
     - Updates next scheduled date based on frequency

  7. weeklyHRReport.js (96 lines)
     - Cron: Monday 9:00 AM IST
     - Aggregates wellness stats by department
     - Calculates trend vs last week
     - Queues weekly_hr_report emails for HR admins

  8. alertEscalation.js (87 lines)
     - Cron: Every 30 minutes
     - Escalates alerts older than 2 hours
     - Queues alert emails to SENIOR_CLINICIAN/COMPANY_ADMIN
     - Creates AuditLog entries

  9. audioCleanup.js (95 lines)
     - Cron: Every 5 minutes
     - Marks audio as deleted if not confirmed within 30min
     - Detects orphaned sessions (>60min)
     - Creates audit log for manual review

  10. consultationReminders.js (168 lines)
      - Cron: Every 15 minutes
      - 60-minute reminder: 55-65min before scheduled time
      - 15-minute reminder: 10-20min before scheduled time
      - Prevents duplicate reminders via remindersSent tracking
      - Sends to both clinician and employee

✓ Health Check Modules (5 files) - /src/checks/
  1. apiCheck.js
     - GET https://api.vocalysis.cittaa.in/v1/health
     - Timeout: 5000ms
     - Status logic: healthy <500ms, degraded 500-2000ms, down >2000ms or non-200

  2. vococoreCheck.js
     - Tries internal Railway URL first, falls back to env var
     - Timeout: 3000ms
     - Detects service health and response time

  3. databaseCheck.js
     - MongoDB admin().ping()
     - Timeout: 3000ms
     - Verifies database connectivity

  4. redisCheck.js
     - Redis PING command
     - Retrieves memory usage stats
     - Returns latency metrics

  5. workerCheck.js
     - Queries Bull queue statistics via Redis
     - Checks: waiting count, active count, failed count
     - Alerts if failed>10 or waiting>100 per queue
